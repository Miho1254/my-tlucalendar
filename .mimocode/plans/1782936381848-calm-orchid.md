# Plan: Normalise all date/time handling to Vietnam timezone (GMT+7)

## Problem

All `DateTime.now()` calls use device-local time. If a user travels abroad (e.g. UTC+9 Japan), the app shows wrong "today", wrong course status, wrong notifications, and wrong survival progress. 99% of users are in Vietnam — the app should always show Vietnam time.

## Scope

- **49 `DateTime.now()` calls** across 14 files
- Notification scheduling, daily summary, course filtering, UI status indicators
- NOT changing: database timestamps (epoch millis, timezone-agnostic), log timestamps, note IDs

## Approach

### Step 1: Create `lib/utils/vn_time.dart` — centralized Vietnam time utility

```dart
import 'package:timezone/timezone.dart' as tz;

/// Always returns the current moment in Vietnam time (Asia/Ho_Chi_Minh).
/// Use this instead of DateTime.now() wherever the app needs "now in Vietnam".
class VnTime {
  static const String _timezoneId = 'Asia/Ho_Chi_Minh';

  /// Current moment in Vietnam timezone
  static DateTime now() {
    final vn = tz.getLocation(_timezoneId);
    return tz.TZDateTime.now(vn);
  }

  /// Today at midnight in Vietnam time
  static DateTime today() {
    final n = now();
    return DateTime(n.year, n.month, n.day);
  }

  /// Convert any DateTime (from DB epoch millis) to Vietnam-local date
  static DateTime fromDateMs(int millisecondsSinceEpoch) {
    final dt = DateTime.fromMillisecondsSinceEpoch(millisecondsSinceEpoch);
    // Epoch millis from DB are already Vietnam time (stored as local)
    // Just strip time component
    return DateTime(dt.year, dt.month, dt.day);
  }

  /// Check if two dates are the same day (ignoring time)
  static bool isSameDay(DateTime a, DateTime b) =>
      a.year == b.year && a.month == b.month && a.day == b.day;
}
```
